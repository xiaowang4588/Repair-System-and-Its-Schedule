## 1. 项目概述

### 1.1 系统简介

多媒体设备报修系统是专为**重庆移通学院綦江校区**设计的教室多媒体设备管理平台，提供以下核心功能：

- **课表查询**：按教室、教师、班级查询课程安排
- **空教室查询**：实时查询可用教室，支持按楼栋、节次筛选
- **设备报修**：教师提交多媒体设备故障报修
- **报修管理**：管理员处理报修工单、分配处理人、记录处理结果
- **数据统计**：报修趋势分析、故障类型统计、楼栋分布等
- **数据大屏**：可视化展示报修数据，支持下钻分析

### 1.2 用户角色

| 角色 | 访问方式 | 主要功能 |
|------|----------|----------|
| **管理员/教师** | 浏览器访问 teacher-app | 管理课表、处理报修、查看统计、管理学生账号 |
| **学生** | Uni-app（H5/小程序） | 查询课表、查询空教室、提交报修、查看个人报修记录 |

### 1.3 技术栈

| 层 | 技术 | 说明 |
|------|------|------|
| 后端 | Python 3.10+ / Flask | RESTful API 服务 |
| 数据库 | SQLite + Peewee ORM | 轻量级，支持 WAL 模式 |
| 教师端 | HTML / CSS / JavaScript | 纯静态单页应用 |
| 学生端 | Uni-app (Vue 3) | 支持 H5 和微信小程序 |
| 数据源 | Excel / 青果教务系统 API | 双数据源，可切换 |

## 4. 安装部署

### 4.1 开发环境部署

#### 步骤 1：克隆项目

```bash
git clone <仓库地址>
cd course_query
```

#### 步骤 2：安装 Python 依赖

```bash
cd api-server
pip install -r requirements.txt
```

**依赖清单**：
| 包 | 版本 | 用途 |
|------|------|------|
| flask | >=3.0.0 | Web 框架 |
| flask-cors | >=4.0.0 | 跨域支持 |
| pandas | >=2.2.0 | 数据处理 |
| openpyxl | >=3.1.0 | Excel 读写 |
| pytz | >=2024.1 | 时区处理 |
| requests | >=2.31.0 | HTTP 请求 |
| peewee | >=4.0.0 | SQLite ORM |

#### 步骤 3：配置环境变量

复制并编辑 `.env` 文件：

```bash
cp .env.example .env
# 编辑 .env，配置数据源类型和教务系统账号（如需 API 模式）
```

#### 步骤 4：启动后端服务

```bash
python app.py
```

服务将在 `http://localhost:5000` 启动。

#### 步骤 5：访问前端

- **教师端**：用浏览器打开 `teacher-app/index.html`
- **学生端**：
  ```bash
  cd student-app
  npm install
  npm run dev:h5
  ```

### 4.2 生产环境部署

#### 方式一：Gunicorn（推荐）

```bash
cd api-server
gunicorn -c ../deploy/gunicorn_conf.py app:app
```

**Gunicorn 配置**（`deploy/gunicorn_conf.py`）：
```python
bind = "0.0.0.0:5000"
workers = 4
threads = 2
timeout = 120
```

#### 方式二：uWSGI

```bash
cd api-server
uwsgi ../deploy/uwsgi.ini
```

#### 方式三：Docker（需自行编写 Dockerfile）

```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY api-server/ .
RUN pip install -r requirements.txt
EXPOSE 5000
CMD ["gunicorn", "-c", "gunicorn_conf.py", "app:app"]
```

#### Nginx 反向代理配置

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # API 服务
    location /api/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /admin/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 教师端
    location /teacher/ {
        alias /path/to/teacher-app/;
        index index.html;
    }

    # 学生端
    location / {
        root /path/to/student-app/dist;
        try_files $uri $uri/ /index.html;
    }
}
```

## 5. 配置说明

### 5.1 环境变量配置（`.env`）

```ini
# 数据源类型：excel 或 api
DATA_SOURCE_TYPE=excel

# 教务系统账号（API 模式必填）
QINGGUO_USERNAME=
QINGGUO_PASSWORD=

# 学期配置
QINGGUO_XNM=2025          # 学年（起始年份）
QINGGUO_XQM=12            # 学期（12=春季，3=秋季）
QINGGUO_XQH_ID=C67E548C4B4553FFE0530100007F06AD  # 校区 ID

# 缓存配置
CACHE_TTL_SECONDS=1800     # 缓存有效期（秒）
CACHE_BACKGROUND_REFRESH=true   # 后台刷新
CACHE_FALLBACK_TO_STALE=true    # 降级兜底
```

### 5.2 运行时配置（`config.json`）

```json
{
    "admin": {
        "username": "wxzx",
        "password_hash": "<SHA-256 哈希值>"
    },
    "datasource": {
        "type": "excel",
        "current_excel": "uploads/2025-2026春季_全校课表.xlsx",
        "api": {
            "base_url": "https://教务系统地址",
            "username": "",
            "password": ""
        }
    },
    "semester": {
        "start_date": "2026-03-02"
    },
    "cache": {
        "ttl_seconds": 1800,
        "background_refresh": true,
        "fallback_to_stale": true
    }
}
```

### 5.3 数据源切换

**方式一：修改 `.env` 文件**
```ini
DATA_SOURCE_TYPE=api  # 或 excel
```

**方式二：通过教师端后台**
1. 登录教师端
2. 进入「数据源配置」
3. 选择「Excel 模式」或「API 模式」
4. 保存配置

### 5.4 学期配置

首次使用需要设置学期开始日期：

1. 登录教师端
2. 进入「周次设置」
3. 选择学期开始日期（通常是第一周周一）
4. 确认保存

系统会自动计算当前是第几教学周。

## 6. 教师端使用指南

### 6.1 登录

1. 用浏览器打开 `teacher-app/index.html`
2. 输入管理员账号和密码
   - 默认账号：`wxzx`
   - 默认密码：`123456`
3. 点击「登录」

> ⚠️ **安全提示**：首次登录后请立即修改默认密码！

### 6.2 课表管理

#### 6.2.1 上传课表 Excel

1. 进入「课表管理」标签
2. 点击「上传课表」
3. 选择 Excel 文件（.xlsx 或 .xls）
4. 等待上传完成

**Excel 格式要求**：

| 列 | 字段名 | 说明 |
|------|--------|------|
| A | 星期几 | 周一、周二... |
| B | 上课节次 | 1-2节、3-4节... |
| C | 上课地点 | 完整教室名称 |
| D | 课程名称 | 课程全称 |
| E | 姓名 | 授课教师 |
| F | 教工号 | 教师工号 |
| G | 开课学院 | 学院名称 |
| H | 教学班组成 | 班级信息 |
| I | 上课时间 | 周次范围 |

#### 6.2.2 切换数据源

1. 进入「数据源配置」
2. 选择数据源类型：
   - **Excel 模式**：从本地文件读取，离线可用
   - **API 模式**：从教务系统实时获取，需配置账号
3. 如选 API 模式，填写教务系统账号密码
4. 点击「测试连接」验证配置
5. 保存配置

### 6.3 报修管理

#### 6.3.1 查看报修列表

1. 进入「报修管理」标签
2. 可使用筛选条件：
   - 状态：未处理 / 处理中 / 已处理
   - 楼栋：选择具体楼栋
   - 时间范围：选择起止日期
   - 关键词：搜索教室、报修人、处理人

#### 6.3.2 新增报修记录

1. 点击「新增报修」按钮
2. 填写表单字段：

| 字段 | 必填 | 说明 |
|------|------|------|
| 故障教室 | ✅ | 教室名称，如「明德楼301」 |
| 报修时间 | - | 默认当前时间，可修改 |
| 周次 | - | 教学周次（1-30） |
| 报修类型 | ✅ | 中控/电脑/投影仪/音响/麦克风/展台/幕布/网络/软件/其他 |
| 报修人 | ✅ | 报修教师姓名 |
| 报修人学院 | - | 教师所属学院 |
| 外聘 | - | 是否外聘教师 |
| 报修方式 | - | 多媒体报修群/电话/其他 |
| 处理人 | ✅ | 负责处理的人员 |
| 更换设备 | - | 是否更换了设备 |
| 处理情况 | - | 未处理/处理中/已处理 |
| 故障原因 | - | 故障描述 |
| 处理方式 | - | 处理方法描述 |
| 备注 | - | 补充说明 |

3. 点击「保存」

#### 6.3.3 编辑报修记录

1. 在报修列表中找到目标记录
2. 点击「编辑」按钮（铅笔图标）
3. 修改需要更新的字段
4. 点击「保存」

> 系统会自动记录修改历史，可在详情中查看修改日志。

#### 6.3.4 删除报修记录

1. 在报修列表中找到目标记录
2. 点击「删除」按钮（垃圾桶图标）
3. 确认删除

> ⚠️ 删除操作不可恢复，请谨慎操作！

#### 6.3.5 批量操作

1. 勾选需要操作的记录（支持全选）
2. 点击批量操作按钮：
   - **批量修改状态**：统一设置为处理中/已处理
   - **批量分配处理人**：统一指定处理人
   - **批量删除**：删除选中的记录

#### 6.3.6 导入导出

**导出报修记录**：
1. 点击「导出 Excel」
2. 选择导出范围（可选筛选条件）
3. 下载 Excel 文件

**导入报修记录**：
1. 点击「导入」
2. 下载导入模板
3. 按模板格式填写数据
4. 上传填写好的 Excel 文件
5. 确认导入

### 6.4 数据统计

#### 6.4.1 统计概览

进入「数据统计」标签，查看：
- 报修总数、待处理数、已处理数
- 故障类型分布（饼图）
- 楼栋报修分布（柱状图）
- 近期报修趋势（折线图）

#### 6.4.2 数据大屏

1. 点击「数据大屏」按钮
2. 全屏展示可视化数据：
   - 实时报修统计
   - 楼栋/学院/处理人/故障类型下钻分析
   - 时间范围筛选（本周/本月/本学期）

### 6.5 学生账号管理

#### 6.5.1 查看学生列表

1. 进入「学生管理」标签
2. 查看所有学生账号
3. 支持按学号、姓名搜索

#### 6.5.2 创建学生账号

**单个创建**：
1. 点击「新增学生」
2. 填写学号、姓名、学院
3. 点击「保存」

**批量创建**：
1. 点击「批量创建」
2. 按格式输入多个学生信息（每行一个：学号,姓名,学院）
3. 点击「创建」

#### 6.5.3 重置学生密码

1. 在学生列表中找到目标学生
2. 点击「重置密码」
3. 确认重置（密码将重置为学号后6位）

### 6.6 系统设置

#### 6.6.1 修改管理员密码

1. 进入「系统设置」
2. 输入当前密码
3. 输入新密码并确认
4. 保存

#### 6.6.2 缓存设置

- **缓存有效期**：设置数据缓存时间（默认 30 分钟）
- **后台刷新**：是否在缓存过期前自动刷新
- **降级兜底**：刷新失败时是否继续使用旧数据

#### 6.6.3 周次设置

1. 选择学期开始日期
2. 系统自动计算当前教学周
3. 可手动确认/调整当前周次

#### 6.6.4 查看系统日志

1. 进入「系统日志」
2. 查看最近的系统操作记录
3. 支持清空日志

## 7. 学生端使用指南

### 7.1 登录

1. 打开学生端应用（H5 或小程序）
2. 输入学号和密码
3. 点击「登录」

> 初始密码为学号后 6 位，首次登录后建议修改。

### 7.2 首页

登录后进入首页，显示：
- 用户信息（学号、姓名）
- 报修统计（我的报修数量、待处理数）
- 快捷操作入口：
  - 提交报修
  - 课表查询
  - 空教室查询
- 最新报修动态

### 7.3 课表查询

#### 7.3.1 按教室查询

1. 进入「课表」标签
2. 选择星期和节次
3. 输入教室名称
4. 点击查询

#### 7.3.2 按课程/教师查询

1. 切换查询方式（课程名称/教师姓名）
2. 输入关键词
3. 查看查询结果

#### 7.3.3 周课表

1. 输入班级名称或教师姓名
2. 选择查询类型（班级/教师）
3. 查看一周完整课表

### 7.4 空教室查询

1. 进入「空教室」页面
2. 选择查询条件：
   - 星期几
   - 节次（可多选）
   - 楼栋（可选）
3. 点击查询
4. 查看可用教室列表

### 7.5 提交报修

#### 7.5.1 快速报修

1. 点击首页「提交报修」
2. 输入故障教室名称
3. 系统自动匹配：
   - 当前课程
   - 授课教师
   - 所属学院
4. 选择故障类型
5. 添加备注和图片（可选）
6. 提交报修

#### 7.5.2 报修字段说明

| 字段 | 说明 |
|------|------|
| 故障教室 | 必填，教室名称 |
| 报修类型 | 必填，选择故障设备类型 |

## 10. 常见问题

### 10.1 安装部署问题

**Q：pip install 报错**

A：尝试使用国内镜像源：
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

**Q：启动报错 "Address already in use"**

A：端口 5000 被占用，修改 `app.py` 中的端口或杀掉占用进程：
```bash
# Linux/Mac
lsof -i :5000
kill -9 <PID>

# Windows
netstat -ano | findstr :5000
taskkill /PID <PID> /F
```

**Q：数据库锁定错误**

A：SQLite WAL 模式下偶尔会出现，通常等待几秒会自动恢复。如频繁出现，检查是否有多个进程同时写入。

### 10.2 数据源问题

**Q：Excel 上传失败**

A：检查以下几点：
1. 文件格式必须是 `.xlsx` 或 `.xls`
2. 文件大小不超过 16MB
3. 文件必须包含必需的列（星期几、上课节次、上课地点等）
4. 工作表名称必须是 `Sheet1`（或在 `.env` 中配置）

**Q：API 模式获取数据失败**

A：检查以下几点：
1. 教务系统账号密码是否正确
2. 教务系统是否可访问（可能需要 VPN）
3. 学期配置是否正确（xnm、xqm）
4. 查看 `api-server/logs/app.log` 获取详细错误信息

**Q：缓存数据不是最新的**

A：尝试以下操作：
1. 登录教师端，点击「刷新缓存」
2. 或调用 API：`POST /admin/refresh`
3. 检查缓存配置，缩短 TTL 时间

### 10.3 报修功能问题

**Q：学生无法登录**

A：检查以下几点：
1. 学号是否已创建账号（教师端学生管理）
2. 密码是否正确（默认为学号后6位）
3. Token 是否过期（24小时有效期）

**Q：报修提交失败**

A：检查以下几点：
1. 教室名称是否填写
2. 报修类型是否选择
3. 网络连接是否正常

**Q：Excel 导入报错**

A：确保 Excel 格式与导出格式一致，可先下载导入模板参照填写。

### 10.4 前端显示问题

**Q：教师端页面空白**

A：检查以下几点：
1. 浏览器控制台是否有错误（F12）
2. API 服务是否启动
3. 跨域配置是否正确（`flask-cors`）

**Q：学生端 H5 无法访问 API**

A：检查 `manifest.json` 中的代理配置：
```json
{
    "h5": {
        "devServer": {
            "proxy": {
                "/api": {
                    "target": "http://localhost:5000"
                }
            }
        }
    }
}
```

# deploy/gunicorn_conf.py
