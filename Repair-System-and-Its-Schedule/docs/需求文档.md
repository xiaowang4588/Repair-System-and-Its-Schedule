## 一、项目概述

### 1.1 项目背景

当前课程查询系统仅支持从 Excel 文件读取数据，数据更新不及时。需要改造为支持**双数据源**（Excel + API）的系统，并增加后台管理功能，方便老师切换数据源和管理配置。

### 1.2 系统角色

| 角色 | 说明 | 使用场景 |
|------|------|----------|
| **学生** | 查询课程、空教室等信息 | 前台页面 |
| **老师/管理员** | 管理系统配置、切换数据源、查看统计 | 后台页面 |

### 1.3 技术架构

#### 2.2.3 课表文件管理（核心功能）

每学期课表不同，教师需要通过后台上传新课表文件。系统不预设默认文件，首次使用必须上传。

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 文件上传 | 上传新学期的 Excel 课表文件 | P0 |
| 学期关联 | 上传时选择对应的学年和学期 | P0 |
| 文件列表 | 显示所有已上传的文件，按学期分组 | P0 |
| 文件切换 | 选择当前使用哪个课表文件 | P0 |
| 文件下载 | 下载已上传的文件 | P1 |
| 文件删除 | 删除旧学期的文件 | P1 |
| 文件验证 | 上传时检查文件格式是否正确 | P0 |

**上传流程**：

```
教师上传课表 → 选择学年/学期 → 验证文件格式 → 保存到服务器 → 自动加载数据
```

## 七、安全要求

1. **后台登录**：密码使用 SHA256 加密存储
2. **Session 安全**：设置合理的 Session 过期时间
3. **文件上传**：限制文件类型（只允许 .xlsx/.xls）和大小
4. **权限控制**：所有 `/admin/*` 路由需要登录才能访问
5. **敏感信息**：API 密码等敏感配置不显示在前端
