> 版本：v1.0
> 更新日期：2026-05-29
> 状态：待开发

---

## 一、项目背景

### 1.1 当前痛点

学校多媒体运维团队的学生在处理教师设备报修时，存在以下问题：

1. **手动填写 Excel 表格**：每次报修都需要打开 Excel 手动填写，效率低
2. **查找课程信息麻烦**：需要根据教室去查当前是哪位老师在上课
3. **数据无法实时同步**：教师端无法实时看到报修情况
4. **统计分析困难**：无法快速统计报修数量、故障类型分布等

### 1.2 解决方案

开发一个在线报修系统，集成到现有的课程查询系统中：

- **学生端**：在线填写报修单，自动关联课程信息，一键导出 Excel
- **教师端**：可视化大屏展示报修数据，实时监控设备状态

## 二、原始 Excel 表格分析

### 2.1 表格结构

**文件名**：`綦江校区设备信息及报修日志记录表.xlsx`

**标题**：綦江校区设备信息及报修日志记录表

**表头（第2行）**：

| 列 | 字段名 | 说明 | 是否必填 |
|----|--------|------|----------|
| A | 序号 | 自增编号 | 自动 |
| B | 故障教室/书院 | 教室名称 | ✅ 必填 |
| C | 报修时间 | 报修日期时间 | ✅ 必填 |
| D | 周次 | 教学周次 | ✅ 必填 |
| E | 报修类型 | 设备故障类型 | ✅ 必填 |
| F | 报修人 | 报修教师姓名（不允许填未知） | ✅ 必填 |
| G | 报修人学院 | 报修教师所属学院 | ✅ 必填 |
| H | 是否为外聘教师 | 是/否 | ✅ 必填 |
| I | 报修方式 | 报修渠道 | ✅ 必填 |
| J | 处理人 | 处理报修的学生姓名 | ✅ 必填 |
| K | 是否更换设备 | 是/否 | ✅ 必填 |
| L | 处理情况 | 处理状态 | ✅ 必填 |
| M | 故障发生原因 | 故障原因描述 | ✅ 必填 |
| N | 处理方式、或未处理原因 | 处理详情 | ✅ 必填 |
| O | 若未处理，处理完毕时间 | 预计完成时间 | 条件必填 |

## 三、功能需求

### 3.1 学生端功能

#### 3.1.1 报修单填写

**填写流程**：
```
选择教室 → 自动查询当前课程/教师 → 自动填充周次 → 学生确认/修改 → 填写故障信息 → 提交
```

**字段说明**：

| 字段 | 填充方式 | 说明 |
|------|----------|------|
| 故障教室 | 手动选择 | 学生从教室列表中选择 |
| 报修时间 | 自动填充 | 系统当前时间，可修改 |
| 周次 | 自动填充 | 从教师端后台读取当前周次，可修改 |
| 报修人 | 自动填充 | **上课教师姓名**（根据教室+时间查询），可修改 |
| 报修人学院 | 自动填充 | **教师所属学院**，可修改 |
| 是否外聘教师 | 手动选择 | 默认"否"，可修改 |
| 报修类型 | 手动选择 | 下拉选择故障类型 |
| 故障原因 | 手动填写 | 文本输入 |
| 处理方式 | 手动填写 | 文本输入 |
| 处理人 | 手动填写 | **学生自己的姓名**（当前登录学生） |

**注意**：
- 报修人 = 上课教师（自动填充，可修改）
- 处理人 = 学生自己（手动填写）
- 周次、报修时间 = 自动填充，可修改

#### 3.1.2 就近空教室推荐

当设备故障无法修复需要更换教室时，系统自动推荐就近空教室。

**推荐逻辑（同楼栋就近原则）**：

```
优先级1：同楼栋同层 → 如 "行者楼408" 推荐 "行者楼401/402/403..."
优先级2：同楼栋不同层 → 如 "行者楼408" 推荐 "行者楼301/501..."
超出同楼栋：不推荐
```

**注意**：只推荐同楼栋的空教室，不跨楼栋推荐。

**匹配条件**：
- 当前时段（节次）必须空闲
- 教室类型匹配（普通教室换普通教室，实验室换实验室）
- 座位数相近（不能换太小的教室）

**功能说明**：

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 自动推荐 | 选择故障教室后自动显示推荐空教室 | P0 |
| 同层优先 | 优先显示同楼栋同层的空教室 | P0 |
| 显示距离 | 显示推荐教室的位置信息 | P1 |
| 一键选择 | 点击推荐教室直接填入换教室字段 | P0 |
| 手动修改 | 可以手动输入其他教室 | P0 |

**界面展示**：

### 4.2 API 接口设计

**报修记录接口**：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/repair/list` | GET | 获取报修列表 |
| `/api/repair/create` | POST | 创建报修记录 |
| `/api/repair/update` | POST | 更新报修记录 |
| `/api/repair/delete` | POST | 删除报修记录 |
| `/api/repair/stats` | GET | 获取报修统计数据 |
| `/api/repair/export` | GET | 导出 Excel 文件（支持时间范围筛选） |

**智能填充接口**：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/repair/auto-fill` | GET | 根据教室自动填充报修信息 |
| `/api/repair/nearby-rooms` | GET | 获取就近空教室推荐 |

**`/api/repair/auto-fill` 参数**：
- `classroom`: 教室名称（如 "行者楼408"）

**返回数据**：
```json
{
    "status": "ok",
    "data": {
        "classroom": "行者楼408",
        "report_time": "2026-05-29 14:30:00",
        "week_number": 14,
        "course_info": {
            "course_name": "高等数学",
            "teacher_name": "张三",
            "teacher_id": "1001",
            "teacher_college": "理学院",
            "class_name": "2025软件工程1班"
        }
    }
}
```

**说明**：
- `week_number`：从教师端后台读取的当前周次
- `course_info`：当前上课信息，仅作参考显示，不填入报修表
- 报修人（学生姓名）和报修人学院（学生学院）由学生手动填写

**`/api/repair/nearby-rooms` 参数**：
- `classroom`: 故障教室名称（如 "行者楼408"）

- `section`: 节次（如 "5-6节"）
